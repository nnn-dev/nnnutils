NAME

launch.sh - launching action

 

SYNOPSIS

launch.sh [options] -l

launch.sh [options] <module> [launch]

 

DESCRIPTION

launch.sh can be use to do an action for modules of an application.

With this script you can manage easily an application using multiples modules.

 

OPTIONS

-M <directory>
indicates the path of the modules files (see MODULES). default : launch-modules

-E <file>
indicates the path of the environment file (see MODULES). default: none

-l
show list of modules

-A nosys
action on all modules without sys prefix. Use it instead <module>.

-A allsys
action on all modules. Use it instead <module>.

-v
show this launch.sh script's version.

-?
show the help message.

 

COMMANDS

launch
launch the action (it's default)

 

MODULES

A module is a file describing how to launch the action. It's a script file with particular format (see below). This file contains shell functions and description. An environment file can be used too for storing parameters or global functions.

 

Environment file

It's an optional script launched on current shell (with . (dot) command). This file must be indicated with the -E option.

 

Filename

This module's file must be in a directory (by default launch-modules ) with one of this format : <number>_<name> or <number>_sys_<name> were
<number>
is a number. It indicates the order to export this module when -A option is used.

<name>
is the name you want.
sys
is used for indicated a system module.

 

System modules

A system module represents a component used globally on the system. Like ldap server or dns server that is when stopping this component has an impact on the machine and on the other applications.

 

Format of the file

The file must contain description of the module (comment with NNNA) and a shell function:
fn_launch

This is the function launched.

Below, a template for the file:


 #!/bin/sh


 # ################################################# #
 # NNNA description describe here the component      #
 # NNNA author      you                              #
 # NNNA version     version of the script            #
 # ################################################# #


 fn_launch()
 {
   return 0 #ok
 }
